## Security Findings

**Dependency CVEs:** None found. All pinned versions current as of 2026-03-16.

### High

| ID | File:Line | Issue | Fix |
|----|-----------|-------|-----|
| H-1 | `api/main.py:369` | `POST /v1/themes` requires zero authentication. Anonymous IPs can flood the gallery. | Require Firebase Auth JWT. Add `max_length=100` to `PublishRequest.name`. |
| H-2 | `api/main.py:435` | **Stored ANSI injection** — `theme.get("name")` written raw into `text/plain` neofetch response. Publisher can embed terminal escape sequences (`\x1b[2J...`) served to any terminal rendering the URL. Directly exploitable via H-1. | Strip non-printable bytes from `name` at publish time (reuse `sanitize_prompt` control-char filter). |
| H-3 | `api/main.py:407` | Download-count endpoint has no per-slug rate limit. Attacker can inflate counts to manipulate gallery ranking. In-process bucket resets on restart. | Add per-IP-per-slug cooldown (1 increment/IP/theme/hr) backed by DB. |
| H-4 | `api/main.py:92` | Non-constant-time string comparison for `/metrics` Bearer token. | Replace with `hmac.compare_digest(auth, f"Bearer {token}")`. |

### Medium

| ID | File:Line | Issue | Fix |
|----|-----------|-------|-----|
| M-1 | `security/ssrf_guard.py:131` | `safe_fetch` passes no `follow_redirects=False`. A redirect from validated public IP to private address bypasses SSRF guard. | Add `follow_redirects=False`; re-validate any `Location` header before following manually. |
| M-2 | `security/ssrf_guard.py:86` | `assert hostname is not None` — stripped by Python `-O`. In security-critical path. | Replace with `if hostname is None: raise ValueError("URL has no hostname")`. |
| M-3 | `api/main.py:271,273,379` | Raw `RuntimeError`/`ValueError` passed as HTTP response detail — leaks provider names, base URLs, LLM output fragments. | Map `RuntimeError` → generic 503 `{"error": "service_unavailable"}`; map validator errors → structured 422 code. |
| M-4 | `api/main.py:321,350,385,423` | `list_themes(limit=1000)` + Python linear scan on every slug-based endpoint. O(N) memory and latency. Timing oracle for slug existence. | Add indexed `slug` column to SQLite and Firestore; query directly. |
| M-5 | `cache/db.py:226` | `increment_download_count` is a non-atomic read-modify-write (TOCTOU). Concurrent downloads lose increments. | `UPDATE themes SET download_count = download_count + 1 WHERE id = ?` (single atomic statement). Mirror with `firestore.Increment(1)`. |
| M-6 | `api/main.py:153` | `PublishRequest.name` and `theme_data` have no length caps. 10 MB name accepted, stored, returned in every list call. | `Annotated[str, Field(max_length=100)]` for name; `max_length=4096` for theme_data. |

### Low / Informational

| ID | Issue |
|----|-------|
| L-1 | In-process token buckets not shared across Cloud Run replicas — documented limitation, requires Redis before multi-replica |
| L-2 | `_is_development()` in `secrets.py` falls back to dev mode if `GCP_PROJECT` unset — startup misconfiguration silently uses `.env` |
| L-3 | `burst_timestamps` double JSON-encoded in Firestore (list serialized to string, stored as Firestore string field) |
| L-4 | `DAILY_GENERATION_LIMIT`/`BURST_WINDOW_SECONDS` read at module import — non-integer env var causes startup crash with no message |
| L-5 | No security response headers (`X-Content-Type-Options`, `X-Frame-Options`, HSTS) — most relevant for `/v1/neofetch` `text/plain` endpoint |
| L-6 | `bandit` scans `.venv/` — exit code 1 from third-party package issues. Add `.venv` to `exclude_dirs` in `pyproject.toml`. |

### Secure patterns praised

- SSRF DNS-rebinding protection via IP substitution before `httpx` call — above-average implementation
- SSRF blocklist covers RFC1918 + CGNAT + IPv4-mapped IPv6 + link-local
- All SQLite queries use parameterised `?` placeholders — no SQL injection
- No unsafe deserialization in project source (no `eval`, `marshal`, or unsafe YAML loading)
- LLM structural output validation blocks injection-driven output
- IP privacy: only `SHA256(IP)[:16]` stored, never raw IP
- Production secrets routed through GCP Secret Manager — no hardcoded keys found
